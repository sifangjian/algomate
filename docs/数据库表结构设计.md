# 算法大陆 - 数据库表结构设计

## 1. 数据库概述

本设计基于 SQLite 数据库，使用 SQLAlchemy ORM 进行操作。数据库表结构设计遵循以下原则：
- 清晰的关系映射
- 完整的字段定义
- 合理的索引设计
- 支持游戏化核心功能

## 2. 表结构设计

### 2.1 笔记表 (notes)

| 字段       | 类型     | 说明                 | 约束           |
| ---------- | -------- | -------------------- | -------------- |
| id         | INTEGER  | 主键                 | PRIMARY KEY    |
| title      | TEXT     | 笔记标题             | NOT NULL       |
| content    | TEXT     | 笔记内容（Markdown） | NOT NULL       |
| npc_id     | INTEGER  | 关联NPC ID           | FOREIGN KEY    |
| created_at | DATETIME | 创建时间             | DEFAULT CURRENT_TIMESTAMP |
| updated_at | DATETIME | 更新时间             | DEFAULT CURRENT_TIMESTAMP |

### 2.2 卡牌表 (cards)

| 字段          | 类型     | 说明                   | 约束           |
| ------------- | -------- | ---------------------- | -------------- |
| id            | INTEGER  | 主键                   | PRIMARY KEY    |
| name          | TEXT     | 卡牌名称（算法技巧名） | NOT NULL       |
| domain        | TEXT     | 所属领域               | NOT NULL       |
| durability    | INTEGER  | 耐久度 0-100           | NOT NULL       |
| note_id       | INTEGER  | 关联笔记ID             | FOREIGN KEY    |
| created_at    | DATETIME | 创建时间               | DEFAULT CURRENT_TIMESTAMP |
| last_reviewed | DATETIME | 最近复习时间           | NULL           |
| is_sealed     | BOOLEAN  | 是否封印（耐久度=0）   | DEFAULT FALSE  |

### 2.3 NPC表 (npcs)

| 字段          | 类型    | 说明           | 约束           |
| ------------- | ------- | -------------- | -------------- |
| id            | INTEGER | 主键           | PRIMARY KEY    |
| name          | TEXT    | NPC名称        | NOT NULL       |
| domain        | TEXT    | 专长领域       | NOT NULL       |
| location      | TEXT    | 所在秘境       | NOT NULL       |
| avatar        | TEXT    | 头像URL        | NULL           |
| system_prompt | TEXT    | 角色设定提示词 | NOT NULL       |

### 2.4 Boss表 (bosses)

| 字段             | 类型    | 说明                    | 约束           |
| ---------------- | ------- | ----------------------- | -------------- |
| id               | INTEGER | 主键                    | PRIMARY KEY    |
| name             | TEXT    | Boss名称                | NOT NULL       |
| difficulty       | TEXT    | 难度等级                | NOT NULL       |
| weakness_domains | TEXT    | 弱点领域（JSON）        | NOT NULL       |
| description      | TEXT    | 描述                    | NOT NULL       |
| question_id      | INTEGER | 关联题目ID              | FOREIGN KEY    |
| source           | TEXT    | leetcode / ai_generated | NOT NULL       |
| drop_rate        | FLOAT   | 掉宝率                  | NOT NULL       |

### 2.5 对话记录表 (dialogue_records)

| 字段             | 类型     | 说明                   | 约束           |
| ---------------- | -------- | ---------------------- | -------------- |
| id               | INTEGER  | 主键                   | PRIMARY KEY    |
| npc_id           | INTEGER  | 关联NPC ID             | FOREIGN KEY    |
| dialogue_content | TEXT     | 对话内容（JSON）       | NOT NULL       |
| generated_cards  | TEXT     | 生成的卡牌列表（JSON） | NOT NULL       |
| created_at       | DATETIME | 创建时间               | DEFAULT CURRENT_TIMESTAMP |

### 2.6 题目表 (questions)

| 字段          | 类型     | 说明       | 约束           |
| ------------- | -------- | ---------- | -------------- |
| id            | INTEGER  | 主键       | PRIMARY KEY    |
| note_id       | INTEGER  | 关联笔记ID | FOREIGN KEY    |
| question_type | TEXT     | 题目类型   | NOT NULL       |
| content       | TEXT     | 题目内容   | NOT NULL       |
| answer        | TEXT     | 参考答案   | NOT NULL       |
| explanation   | TEXT     | 详细解析   | NOT NULL       |
| created_at    | DATETIME | 创建时间   | DEFAULT CURRENT_TIMESTAMP |

### 2.7 答题记录表 (answer_records)

| 字段        | 类型     | 说明        | 约束           |
| ----------- | -------- | ----------- | -------------- |
| id          | INTEGER  | 主键        | PRIMARY KEY    |
| boss_id     | INTEGER  | 关联Boss ID | FOREIGN KEY    |
| card_id     | INTEGER  | 使用的卡牌  | FOREIGN KEY    |
| user_answer | TEXT     | 用户答案    | NOT NULL       |
| is_correct  | BOOLEAN  | 是否正确    | NOT NULL       |
| feedback    | TEXT     | Agent反馈   | NULL           |
| answered_at | DATETIME | 答题时间    | DEFAULT CURRENT_TIMESTAMP |

### 2.8 用户设置表 (user_settings)

| 字段       | 类型     | 说明              | 约束           |
| ---------- | -------- | ----------------- | -------------- |
| id         | INTEGER  | 主键              | PRIMARY KEY    |
| key        | TEXT     | 设置项key（唯一） | UNIQUE NOT NULL |
| value      | TEXT     | 设置值            | NOT NULL       |
| updated_at | DATETIME | 更新时间          | DEFAULT CURRENT_TIMESTAMP |

## 4. 常用设置项

| key               | 说明             | 可选值               | 默认值   |
| ----------------- | ---------------- | -------------------- | -------- |
| game_difficulty   | 游戏难度         | easy / normal / hard | normal   |
| smtp_host         | 邮件SMTP服务器   |                      |          |
| smtp_port         | 邮件SMTP端口     |                      | 587      |
| smtp_username     | 邮件账号         |                      |          |
| smtp_password     | 邮件密码         |                      |          |
| daily_review_time | 每日复习提醒时间 | HH:MM 格式           | 09:00    |
| api_key           | AI模型API Key    |                      |          |

## 5. 索引设计

| 表名           | 索引字段             | 索引类型  | 说明                 |
| -------------- | -------------------- | --------- | -------------------- |
| cards          | domain, durability   | 复合索引  | 加速按领域和耐久度查询 |
| cards          | last_reviewed        | 普通索引  | 加速遗忘曲线计算     |
| answer_records | boss_id, is_correct  | 复合索引  | 加速战斗记录查询     |
| user_settings  | key                  | 唯一索引  | 加速设置项查询       |

## 6. 数据迁移建议

1. **初始化数据**：
   - 预设8个NPC数据
   - 预设基础卡牌数据
   - 预设初始设置项

2. **数据备份**：
   - 定期备份SQLite数据库文件
   - 支持数据导出为JSON格式

3. **版本控制**：
   - 使用SQLAlchemy的迁移工具管理数据库版本
   - 记录数据库结构变更历史

## 7. 性能优化

1. **查询优化**：
   - 为频繁查询的字段创建索引
   - 使用批量操作减少数据库交互

2. **存储优化**：
   - 合理设置字段类型和长度
   - 定期清理过期数据

3. **缓存策略**：
   - 缓存热点数据（如卡牌列表）
   - 使用内存缓存减少数据库查询

## 8. 安全考虑

1. **数据安全**：
   - 敏感信息（如API Key）加密存储
   - 不上传用户数据到云端

2. **输入验证**：
   - 对所有用户输入进行验证
   - 防止SQL注入攻击

3. **权限控制**：
   - 本地数据库访问权限控制
   - 防止未授权访问

# 算法大陆 - 需求分析文档

## 1. 产品概述

### 1.1 产品背景

算法学习是一个长期的、枯燥的过程，容易中途放弃、前学后忘。用户需要一个游戏化的学习助手，将算法学习转化为一场冒险旅程，在收集卡牌、挑战Boss的过程中对抗遗忘，真正掌握常用算法。

### 1.2 产品定位

一款基于AI Agent的游戏化算法学习助手，核心目标是帮助用户**对抗遗忘曲线**，通过卡牌收集、Boss挑战、遗忘曲线复习等机制，将零散的算法知识逐步构建成系统化的个人知识体系。

### 1.3 核心价值

- **游戏化学习**: 卡牌收集、Boss挑战、等级成长让学习不再枯燥
- **对抗遗忘**: 基于遗忘曲线理论，提供科学的定期复习机制
- **AI导师**: 通过对话问答学习算法技巧，AI自动分析产出卡牌
- **个性化出题**: AI根据用户薄弱点针对性生成Boss

### 1.4 目标用户

面向所有想系统学习算法和知识的人群，尤其适合：
- 面试准备者
- 算法初学者
- 需要持续巩固算法的工程师

## 3. 功能模块详细说明

### 3.1 卡牌系统

#### 3.1.1 卡牌本质

**卡牌 = 算法技巧**

每张卡牌代表一个独立的算法技巧，如「二分查找」「滑动窗口」「DFS」等。

#### 3.1.2 卡牌属性

| 属性         | 说明                                    |
| ------------ | --------------------------------------- |
| 名称         | 算法技巧名称                            |
| 所属领域     | 所属算法分类（新手法森林、迷雾沼泽等）  |
| 耐久度       | 0-100，低于30为"濒危"，系统优先推送复习 |
| 关联笔记     | 产出这张卡牌的源笔记                    |
| 创建时间     | 用于遗忘曲线计算                        |
| 最近复习时间 | 上次复习时间                            |

#### 3.1.3 耐久度规则

| 行为           | 耐久度变化                |
| -------------- | ------------------------- |
| 初始创建       | 80                        |
| 复习/击败Boss  | min(100, 耐久度 + 20)     |
| 未击败Boss     | max(0, 耐久度 - 5)        |
| 未复习（每天） | max(0, 耐久度 - 2)        |
| 濒危阈值       | 耐久度 < 30               |
| 卡牌消散       | 耐久度 = 0 时进入封印状态 |

#### 3.1.4 卡牌获取途径

| 途径        | 说明                                            |
| ----------- | ----------------------------------------------- |
| NPC对话学习 | 与领域专家NPC对话，一问一答，记录笔记，产出卡牌 |
| Boss掉落    | 击败Boss后，有概率从知识宝库中解锁新卡牌        |

### 3.2 NPC对话系统

#### 3.2.1 NPC列表

| NPC            | 领域         | 所在秘境 | 专长                 |
| -------------- | ------------ | -------- | -------------------- |
| **老夫子**     | 基础数据结构 | 新手森林 | 数组、链表、栈、队列 |
| **树语者**     | 树与图       | 新手森林 | 二叉树、图的遍历     |
| **沼泽向导**   | 搜索与遍历   | 迷雾沼泽 | DFS、BFS、拓扑排序   |
| **圣殿智者**   | 动态规划     | 智慧圣殿 | 状态设计、DP优化     |
| **贪婪之王**   | 贪心算法     | 贪婪之塔 | 贪心选择证明         |
| **迷宫守护者** | 回溯算法     | 命运迷宫 | 剪枝、枚举           |
| **分裂贤者**   | 分治算法     | 分裂山脉 | 递归设计             |
| **数学巫师**   | 数学与位运算 | 数学殿堂 | 位运算、数学技巧     |

#### 3.2.2 对话流程

```
进入秘境 → 找到NPC → 选择学习话题 → 开始对话
    ↓
[气泡UI，一问一答]
    ↓
用户提问 or NPC讲解 → 记录笔记 → 提交笔记
    ↓
对话结束 → AI分析笔记 → 产出卡牌
```

### 3.3 Boss挑战系统

#### 3.3.1 Boss属性

| 属性     | 说明                         |
| -------- | ---------------------------- |
| 名称     | Boss名称（如"迷雾史莱姆王"） |
| 难度     | 简单/中等/困难               |
| 弱点     | 克制它的卡牌类型             |
| 关联卡牌 | 击败它能提升耐久度的卡牌     |
| 描述     | 题目背景故事（游戏化包装）   |
| 掉宝率   | 击败后掉落新卡牌的概率       |

#### 3.3.2 Boss生成规则

1. **遗忘曲线触发**: 系统根据艾宾浩斯曲线推送需要复习的Boss
2. **基于现有卡牌**: Boss的解题所需技巧必须在用户已拥有的卡牌中
3. **个性化难度**: 根据用户历史表现动态调整
4. **不超出能力**: 新用户只会遇到使用已掌握技巧的Boss

#### 3.3.3 Boss战流程

```
遭遇Boss → 显示Boss描述 → 选择应战卡牌
    ↓
进入答题界面 → 完成答题 → 判定结果
    ↓
[成功] 击败Boss → 卡牌耐久度提升 → 经验/奖励/掉宝
[失败] Boss逃脱 → 耐久度下降
    ↓
继续冒险 或 传送回家
```

### 3.4 遗忘曲线系统

#### 3.4.1 复习节点

基于艾宾浩斯遗忘曲线，设置以下复习节点：

```
创建卡片 → 1天后 → 3天后 → 7天后 → 14天后 → 30天后 → 60天后
```

#### 3.4.2 复习任务生成

- 每天生成「每日修炼」任务列表
- 优先推送濒危卡牌（耐久度 < 30）
- 结合长期遗忘曲线和Boss挑战
- 用户可选择修炼或跳过

#### 3.4.3 复习形式

| 形式     | 说明                                |
| -------- | ----------------------------------- |
| 笔记翻阅 | 查看卡牌关联的笔记内容              |
| 快速问答 | AI生成1-2道相关概念的选择题或填空题 |
| Boss挑战 | 使用该技巧的Boss战                  |

### 3.5 秘境解锁系统

#### 3.5.1 解锁规则

算法大陆不会一次性开放所有秘境。新秘境/NPC的解锁需要满足"熟练度达标卡牌数量"要求：

| 秘境         | 解锁条件                      | 包含NPC           |
| ------------ | ----------------------------- | ----------------- |
| **新手森林** | 默认解锁                      | 老夫子、树语者    |
| **迷雾沼泽** | 新手森林熟练度≥60的卡牌 ≥ 3张 | 沼泽向导          |
| **智慧圣殿** | 迷雾沼泽熟练度≥60的卡牌 ≥ 3张 | 圣殿智者          |
| **贪婪之塔** | 智慧圣殿熟练度≥60的卡牌 ≥ 3张 | 贪婪之王          |
| **命运迷宫** | 贪婪之塔熟练度≥60的卡牌 ≥ 3张 | 迷宫守护者        |
| **分裂山脉** | 命运迷宫熟练度≥60的卡牌 ≥ 3张 | 分裂贤者          |
| **数学殿堂** | 分裂山脉熟练度≥60的卡牌 ≥ 3张 | 数学巫师          |
| **试炼之地** | 所有领域熟练度≥60的卡牌 ≥ 5张 | 无NPC，纯Boss挑战 |

**熟练度达标定义**: 耐久度 ≥ 60

### 3.6 游戏难度系统

游戏难度影响以下参数：

| 难度等级 | 耐久度变化率 | Boss掉卡率 | 每日任务数量 |
| -------- | ------------ | ---------- | ------------ |
| **简单** | 降低50%      | +15%       | 3个          |
| **普通** | 标准         | 标准       | 5个          |
| **困难** | 增加50%      | +30%       | 8个          |

## 5. 非功能性需求

### 5.1 性能需求
- 页面加载时间 < 2秒
- 邮件发送成功率 > 99%
- AI响应时间 < 5秒

### 5.2 安全需求
- 不上传用户数据到云端
- API Key等敏感信息本地加密存储

### 5.3 可扩展性
- 支持插件机制扩展功能
- 支持自定义秘境/NPC
- 支持数据导出/导入

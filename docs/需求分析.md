# 算法学习助手 - 需求分析文档

## 1. 产品概述

### 1.1 产品背景

我们在学习算法过程中常常面临"前学后忘"的痛点，随着时间推移知识点逐渐模糊，需要一个智能化的学习助手来帮助整理、巩固和强化学习成果，将零散的知识点系统化、结构化。

### 1.2 产品定位

一款基于AI Agent的个人算法学习助手，核心目标是帮助用户对抗遗忘曲线，通过笔记整理、定期复习、智能出题等方式，将零散的算法知识逐步构建成系统化的个人知识体系。

### 1.3 核心价值

- **整理归类**：将用户碎片化的学习笔记自动整理成体系
- **对抗遗忘**：基于遗忘曲线理论，提供科学的定期复习提醒
- **查漏补缺**：通过智能出题测试，精准定位薄弱点并针对性训练
- **可视化管理**：游戏化进度展示，增强学习动力

### 1.4 目标用户

- **初期**：个人用户（自己使用）
- **长期**：面向所有想系统学习算法和知识的人群

## 2. 用户需求详细说明

### 2.1 核心功能需求

| 优先级 | 功能模块   | 需求描述                                                             |
| ------ | ---------- | -------------------------------------------------------------------- |
| P0     | 笔记管理   | 用户以Markdown格式输入/粘贴学习笔记，系统自动解析并存储              |
| P0     | 知识归类   | 按算法类型自动分类（如DFS、BFS、动态规划、贪心等），其他维度作为标签 |
| P0     | 复习提醒   | 通过邮件定时发送当日复习内容                                         |
| P1     | 智能出题   | 根据薄弱点生成混合形式练习题（代码题、选择题、简答题）               |
| P1     | 薄弱点分析 | 基于答题情况自动分析用户薄弱知识点                                   |
| P2     | 进度可视化 | 展示学习进度、掌握程度雷达图、遗忘曲线等                             |
| P2     | 知识图谱   | 可视化个人知识体系结构                                               |
| P3     | 多知识领域 | 支持操作系统等其他知识领域的学习                                     |

### 2.2 使用场景

| 场景  | 描述                                                 |
| ----- | ---------------------------------------------------- |
| 场景1 | 用户在电脑上编写Markdown笔记，通过软件导入或粘贴内容 |
| 场景2 | 用户从网络摘录优质算法文章片段粘贴到软件中           |

| 场景5 | 用户完成复习后，进行答题测试检验掌握程度             |
| 场景6 | Agent根据用户答题情况分析薄弱点，生成针对性练习      |

## 3. 功能模块详细说明

### 3.1 笔记管理模块

#### 3.1.1 笔记输入
- 支持直接输入Markdown格式笔记
- 支持复制粘贴已有笔记内容
- 支持从外部Markdown文件导入

#### 3.1.2 笔记解析
- 自动提取笔记中的代码片段
- 自动识别题目描述、解题思路、复杂度分析等
- 自动关联相关算法知识点

#### 3.1.3 笔记存储
- 本地SQLite数据库存储
- 笔记数据结构包含：标题、内容、创建时间、修改时间、关联题目、掌握程度等字段

### 3.2 知识归类模块

#### 3.2.1 分类维度
- **主分类**：按算法类型分类
  - 基础数据结构（数组、链表、栈、队列、哈希表）
  - 树与图
  - 排序算法
  - 搜索与遍历（DFS、BFS）
  - 动态规划
  - 贪心算法
  - 回溯算法
  - 分治算法
  - 数学与位运算
  - 其他

- **辅助标签**（多选）
  - 题目难度：简单、中等、困难
  - 学习进度：已掌握、模糊、薄弱、未接触
  - 来源：LeetCode、代码随想录、自定义等
  - 时间：首次学习日期、最近复习日期

#### 3.2.2 智能分类
- 基于AI大模型理解笔记内容
- 自动判断算法类型并分类
- 自动提取关键标签

### 3.3 复习提醒模块

#### 3.3.1 提醒机制
- 基于遗忘曲线理论（艾宾浩斯遗忘曲线）安排复习时间
- 复习节点：1天后、3天后、7天后、14天、30天、60天

#### 3.3.2 邮件通知
- 定时发送邮件（可配置发送时间）
- 邮件内容包含：复习知识点列表、关联题目、复习建议
- 支持配置SMTP邮件服务器

#### 3.3.3 复习任务管理
- 展示待复习内容列表
- 支持标记已完成/已掌握
- 支持跳过本次复习

### 3.4 智能出题模块

#### 3.4.1 题目来源
- 基于用户笔记中的题目
- 基于用户薄弱点自动生成
- 支持手动指定出题范围

#### 3.4.2 题目类型
- **选择题**：概念题、特点题、时间复杂度分析题
- **简答题**：算法思路解释、复杂度分析、边界条件讨论
- **代码题**：代码补全、输出预测、bug查找

#### 3.4.3 出题策略
- 优先推送薄弱知识点相关题目
- 混合多种题型
- 根据用户表现动态调整难度

### 3.5 薄弱点分析模块

#### 3.5.1 分析维度
- 按算法类型统计正确率
- 识别高频错误类型（如边界条件处理、复杂度计算等）
- 追踪遗忘趋势

#### 3.5.2 分析报告
- 生成个人薄弱点报告
- 提供针对性学习建议
- 动态调整复习计划

### 3.6 进度可视化模块

#### 3.6.1 学习进度
- 总学习天数
- 笔记数量统计
- 题目完成数量

#### 3.6.2 掌握程度
- 雷达图展示各算法类型掌握程度
- 进度条展示整体学习进度
- 等级系统（游戏化）

#### 3.6.3 学习曲线
- 遗忘曲线图
- 学习效率趋势图
- 薄弱点变化趋势

### 3.7 Agent核心模块

#### 3.7.1 Agent能力
- **笔记理解**：理解用户笔记内容，提取关键知识点
- **智能分类**：判断笔记应属的算法类型和标签
- **题目生成**：根据薄弱点生成针对性练习题
- **答题评判**：分析用户答案，给出评价和改进建议
- **学习规划**：根据用户情况制定复习计划

#### 3.7.2 Agent交互
- 支持多轮对话交互
- 支持追问和深入讨论
- 支持个性化学习建议

## 5. 邮件提醒功能

### 5.1 邮件配置
- 用户配置SMTP服务器信息（地址、端口、账号、密码）
- 支持主流邮箱服务（QQ邮箱、网易邮箱、Gmail等）
- 支持TLS/SSL加密

### 5.2 邮件内容模板

邮件内容分为两部分：**复习知识列表** + **详细知识内容**

#### 5.2.1 复习知识列表（概览）
```
主题：📚 算法学习助手 - 今日复习提醒

亲爱的学习者，您好！

今天是复习日，以下是您需要复习的内容：

【动态规划】
📝 背包问题 - 完全背包
📝 最长公共子序列

📊 今日复习目标：2个知识点
⏰ 预计用时：15分钟
```

#### 5.2.2 详细知识内容（碎片时间复习）
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📖 知识点：完全背包问题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【核心思路】
完全背包是在01背包基础上，每种物品可以无限次选取。
状态定义：dp[i][j] 表示前i种物品放入容量为j的背包能获得的最大价值
状态转移：dp[i][j] = max(dp[i-1][j], dp[i][j-weight[i]] + value[i])

【代码模板】
for i in range(1, n+1):
    for j in range(w, capacity+1):
        dp[j] = max(dp[j], dp[j-weight[i]] + value[i])

【与01背包区别】
- 01背包：内循环倒序遍历（保证每种物品只选一次）
- 完全背包：内循环正序遍历（允许物品重复选择）

【经典例题】
LeetCode 322. 零钱兑换

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**邮件推送策略**：
- 邮件标题包含当日复习知识点数量和预计用时
- 支持在邮件客户端直接浏览详细内容
- 重点公式和代码使用等宽字体突出显示
- 适合地铁、午休等碎片时间完成复习

# 算法大陆 - 产品设计文档

## 1. 产品愿景

**Slogan**: 在算法大陆的冒险中，成为算法大师

**核心理念**: 算法学习是一场漫长的冒险，而非枯燥的任务列表。用户不是在做题，而是在「算法大陆」上收集卡牌、挑战Boss、对抗遗忘，最终成为掌握算法技巧的冒险者。

**用户价值**:
- **对抗遗忘**: 基于遗忘曲线理论，让用户科学地复习，将短期记忆转化为长期记忆
- **游戏化动力**: 卡牌收集、Boss挑战、耐久度机制让学习不再枯燥
- **个性化学习**: AI导师根据用户薄弱点针对性出题，每一步都走在用户的"最近发展区"

**衡量成功的唯一指标**: 用户是否在持续的冒险过程中真正掌握了常用算法（而非产出笔记数量）

## 2. 世界观设定

### 2.1 算法大陆

用户是一名冒险者，在一片名为「算法大陆」的奇幻世界中历练。这片大陆由无数「算法秘境」组成，每个秘境都蕴含着不同的算法奥秘。

```
算法大陆
├── 新手森林（基础数据结构）
├── 迷雾沼泽（搜索与遍历）
├── 智慧圣殿（动态规划）
├── 贪婪之塔（贪心算法）
├── 命运迷宫（回溯算法）
├── 分裂山脉（分治算法）
├── 数学殿堂（数学与位运算）
└── 试炼之地（综合挑战）
```

### 2.2 世界构成元素

| 元素         | 描述                                |
| ------------ | ----------------------------------- |
| **秘境入口** | 进入特定算法领域的门户              |
| **知识之泉** | NPC所在的安全区域，可与导师对话学习 |
| **Boss巢穴** | 挑战Boss的地点                      |
| **卡牌工坊** | 管理/查看已收集卡牌的场所           |
| **冒险日志** | 查看历史战斗记录、统计信息          |

### 2.3 冒险者成长路径

```
Lv.1 冒险者（新用户）
    ↓ [完成初始任务，获取第一批卡牌]
Lv.5 探索者（掌握基础数据结构）
    ↓ [挑战初级Boss，构建算法知识体系]
Lv.10 修行者（掌握核心算法）
    ↓ [攻克高级Boss，查漏补缺]
Lv.20 算法大师（全面掌握）
    ↓ [持续修炼，保持状态，挑战极限]
Lv.? 至尊大师（持续目标）

### 2.4 秘境解锁机制

#### 2.4.1 解锁规则

算法大陆不会一次性开放所有秘境。新秘境/NPC的解锁需要满足**"熟练度达标卡牌数量"**要求：

| 秘境         | 解锁条件                      | 包含NPC           |
| ------------ | ----------------------------- | ----------------- |
| **新手森林** | 默认解锁                      | 老夫子、树语者    |
| **迷雾沼泽** | 新手森林熟练度≥60的卡牌 ≥ 3张 | 沼泽向导          |
| **智慧圣殿** | 迷雾沼泽熟练度≥60的卡牌 ≥ 3张 | 圣殿智者          |
| **贪婪之塔** | 智慧圣殿熟练度≥60的卡牌 ≥ 3张 | 贪婪之王          |
| **命运迷宫** | 贪婪之塔熟练度≥60的卡牌 ≥ 3张 | 迷宫守护者        |
| **分裂山脉** | 命运迷宫熟练度≥60的卡牌 ≥ 3张 | 分裂贤者          |
| **数学殿堂** | 分裂山脉熟练度≥60的卡牌 ≥ 3张 | 数学巫师          |
| **试炼之地** | 所有领域熟练度≥60的卡牌 ≥ 5张 | 无NPC，纯Boss挑战 |

**熟练度达标定义**: 耐久度 ≥ 60

#### 2.4.2 地图展示

未解锁的秘境在地图上显示为**灰暗区域**，鼠标悬停显示解锁条件。用户可以看到目标，但需要通过学习和挑战来解锁。

```
算法大陆地图

  [新手森林] ←──── 默认解锁
       ↓
  [迷雾沼泽] ←──── 需要3张熟练度≥60的卡牌（基础数据结构）
       ↓
  [智慧圣殿] ←──── 需要3张熟练度≥60的卡牌（搜索与遍历）
       ...
```

#### 2.4.3 解锁提示

当用户满足解锁条件时：
- 地图上灰暗秘境逐渐**点亮**
- 播放解锁动画
- 显示解锁奖励（新NPC登场）

## 3. 核心概念

### 3.1 卡牌系统

#### 3.1.1 卡牌本质

**卡牌 = 算法技巧**

每张卡牌代表一个独立的算法技巧，如：
- 「二分查找」
- 「滑动窗口」
- 「深度优先搜索 DFS」
- 「广度优先搜索 BFS」
- 「完全背包」
- 「状态压缩DP」

#### 3.1.2 卡牌属性

| 属性         | 说明                                    |
| ------------ | --------------------------------------- |
| **名称**     | 算法技巧名称                            |
| **所属领域** | 所属算法分类                            |
| **耐久度**   | 0-100，低于30为"濒危"，系统优先推送复习 |
| **关联笔记** | 产出这张卡牌的源笔记                    |
| **创建时间** | 用于遗忘曲线计算                        |
| **最近复习** | 上次复习时间                            |

#### 3.1.3 耐久度规则

```
初始创建: 耐久度 = 80
复习/击败Boss: 耐久度 = min(100, 耐久度 + 20)
未击败Boss: 耐久度 = max(0, 耐久度 - 5)
未复习（每天）: 耐久度 = max(0, 耐久度 - 2)

濒危阈值: 耐久度 < 30
卡牌消散: 耐久度 = 0 时，卡牌进入"封印"状态，需通过复习解封
```

#### 3.1.4 卡牌获取途径

| 途径            | 说明                                                                 |
| --------------- | -------------------------------------------------------------------- |
| **NPC对话学习** | 与领域专家NPC对话，一问一答，记录笔记，笔记产出相关卡牌              |
| **Boss掉落**    | 击败Boss后，有概率从Boss的知识宝库中解锁新卡牌（掉率受游戏难度影响） |

### 3.2 NPC系统

#### 3.2.1 NPC定位

NPC是算法大陆上各领域的Expert导师，用户通过与NPC对话来学习特定技巧，对话结束后自动生成对应卡牌。

#### 3.2.2 NPC列表

| NPC            | 领域         | 所在秘境 | 专长                 |
| -------------- | ------------ | -------- | -------------------- |
| **老夫子**     | 基础数据结构 | 新手森林 | 数组、链表、栈、队列 |
| **树语者**     | 树与图       | 新手森林 | 二叉树、图的遍历     |
| **沼泽向导**   | 搜索与遍历   | 迷雾沼泽 | DFS、BFS、拓扑排序   |
| **圣殿智者**   | 动态规划     | 智慧圣殿 | 状态设计、DP优化     |
| **贪婪之王**   | 贪心算法     | 贪婪之塔 | 贪心选择证明         |
| **迷宫守护者** | 回溯算法     | 命运迷宫 | 剪枝、枚举           |
| **分裂贤者**   | 分治算法     | 分裂山脉 | 递归设计             |
| **数学巫师**   | 数学与位运算 | 数学殿堂 | 位运算、数学技巧     |

#### 3.2.3 NPC对话流程

```
进入秘境 → 找到NPC → 选择学习话题 → 开始对话
    ↓
[气泡UI，一问一答]
    ↓
用户提问 or NPC讲解 → 记录笔记 → 提交笔记
    ↓
对话结束 → 系统分析笔记 → 产出卡牌
```

**对话示例**:
```
老夫子: "旅人，看你远道而来，可是向我学习技术的，我掌握有二分查找、线性查找等技巧，你想学哪一个呀？"

用户: "我想学什么是二分查找"

老夫子: "二分查找是一种在有序数组中查找目标元素的算法。
        它的核心思想是'分而治之'：每次将搜索范围缩小一半。
        
        你知道为什么它比线性查找快吗？"

用户: "因为每次排除一半的数据？"

老夫子: "完全正确！假设数组有n个元素，线性查找最坏需要O(n)次，
        而二分查找只需要O(log n)次。
        
        现在让我考考你：如果数组是 [1,3,5,7,9]，要查找5，
        二分查找第一次会比较哪个元素？"

用户: "中间的元素，即7，因为5 < 7，所以会在左半边继续找"

老夫子: "很好！你已经理解了二分的核心思想。
        
        现在记住这个技巧的要点：
        1. 数组必须有序
        2. 维护左右边界 lo 和 hi
        3. 每次取中点，比较后缩小范围
        
        "

记录笔记...

保存笔记 -> AI分析 -> [获得卡牌: 二分查找]
```

### 3.3 Boss系统

#### 3.3.1 Boss本质

Boss是基于用户已有卡牌生成的算法题目。用户通过解决题目来"击败"Boss，同时巩固相关卡牌的耐久度。

#### 3.3.2 Boss属性

| 属性         | 说明                               |
| ------------ | ---------------------------------- |
| **名称**     | Boss名称（如"迷雾沼泽的史莱姆王"） |
| **难度**     | 简单/中等/困难                     |
| **弱点**     | 克制它的卡牌类型（如DFS、BFS）     |
| **关联卡牌** | 击败它能提升耐久度的卡牌           |
| **描述**     | 题目背景故事（游戏化包装）         |

#### 3.3.3 Boss生成规则

1. **遗忘曲线触发**: 系统根据艾宾浩斯曲线，在适当时间推送需要复习的Boss
2. **基于现有卡牌**: Boss的解题所需技巧必须在用户已拥有的卡牌中
3. **个性化难度**: 根据用户历史表现动态调整
4. **不超出能力**: 新用户只会遇到使用已掌握技巧的Boss

#### 3.3.4 Boss战流程

```
遭遇Boss → 显示Boss描述 → 选择应战卡牌
    ↓
进入答题界面 → 完成答题 → 判定结果
    ↓
[成功] 击败Boss → 卡牌耐久度提升 → 经验/奖励
[失败] Boss逃脱 → 耐久度下降
    ↓
继续冒险 或 传送回家
```

**Boss战示例**:
```
=======================================
      🐉 遭遇 Boss: 迷雾史莱姆王
=======================================

难度: ★★☆ 中等
描述: "这只史莱姆居住在一段无序的数组中，
      只有用正确的算法才能将它驱逐！"

----------------------------------------
题目: 给定一个数组 nums 和一个整数 k，
      找出数组中最长的连续子数组，
      使得该子数组的元素和等于 k。

示例: nums = [1,1,1,2], k = 3
      输出: 2 (子数组 [1,1] 或 [1,2] 之一)

时间限制: 30分钟
尝试次数: 3次
掉宝率: 20%

----------------------------------------
[你选择使用的卡牌]

  ┌──────────┐  ┌──────────┐
  │双指针    │  │滑动窗口  │  ← 用户选中
  │ 耐久: 85 │  │ 耐久: 60 │
  └──────────┘  └──────────┘

         [迎战！]
```

### 3.4 遗忘曲线系统

#### 3.4.1 复习节点

基于艾宾浩斯遗忘曲线，设置以下复习节点：

```
创建卡片 → 1天后 → 3天后 → 7天后 → 14天后 → 30天后 → 60天后
```

#### 3.4.2 复习任务生成

- 每天生成「每日修炼」任务列表
- 优先推收濒危卡牌（耐久度 < 30）
- 结合长期遗忘曲线和Boss挑战
- 用户可选择修炼或跳过

#### 3.4.3 复习形式

| 形式         | 说明                                |
| ------------ | ----------------------------------- |
| **笔记翻阅** | 查看卡牌关联的笔记内容              |
| **快速问答** | AI生成1-2道相关概念的选择题或填空题 |
| **Boss挑战** | 使用该技巧的Boss战                  |

## 4. 核心功能流程

### 4.1 用户旅程图

```
┌─────────────────────────────────────────────────────────────┐
│                        新用户引导                             │
│  创建角色 → 选择主修领域 → 领取初始卡组 → 进入冒险                                    │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                        日常冒险循环                           │
│                                                              │
│   ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐   │
│   │ 每日任务 │ → │ 挑战Boss │ → │ 查看结果 │ → │ 复习卡牌 │   │
│   │ (系统推送)│    │ (遗忘曲线)│    │ (获得奖励)│    │ (提升耐久)│   │
│   └─────────┘    └─────────┘    └─────────┘    └─────────┘   │
│                                                              │
│   [持续循环，对抗遗忘]                                         │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                        技能学习                               │
│                                                              │
│ 前往秘境 → 找到NPC → 开始对话 → 一问一答 → 记录冒险者日记 → 获得新卡牌 │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 关键场景流程

#### 4.2.1 对话学习流程

```
1. 用户进入特定秘境
2. 点击NPC头像，进入对话界面
3. NPC打招呼，展示可学习的话题列表
4. 用户选择话题（或自由提问）
5. 开始一问一答对话（气泡UI），记录笔记
6. 提交笔记，对话结束
7. AI检查并润色笔记，分析产生卡牌
8. 用户获得新卡牌，可查看详情，修改内容
```

#### 4.2.2 Boss挑战流程

```
1. 系统推送Boss（遗忘曲线触发 or 用户主动挑战）
2. 显示Boss信息和弱点提示（默认不显示）
3. 用户从自己的卡牌中选择一张/多张应战
4. 进入答题界面：
   - 显示题目描述
   - 用户提交代码或选择答案
   - 系统评判
5. 答题结束，显示结果：
   - 成功：庆祝动画 + 奖励 + 耐久度提升
   - 失败：分析错因 + 耐久度下降 + 推送类似Boss或回家
6. 战斗记录保存
```

#### 4.2.3 主动选卡对战

```
1. 用户点击"修炼"按钮
2. 系统展示用户的卡牌列表（按领域分类）
3. 用户选择一张卡牌进行修炼
4. 系统根据卡牌技巧生成对应Boss
5. 进入Boss挑战流程

首页/冒险地图

│   ├── 进入秘境
│   │   ├── NPC对话界面（包含日记本用来记笔记）
│   │   └── Boss遭遇界面
│   │
│   └── 每日修炼
│       ├── 笔记复习
│       ├── 快速问答

│

### 5.2 核心界面描述

#### 5.2.1 首页/冒险地图

**布局**:
- 顶部: 冒险者等级、昵称、经验值条
- 中间: 算法大陆地图（可点击进入各秘境）
- 底部: 每日任务快捷入口、通知入口

**交互**:
- 点击秘境进入
- 每日任务红点提示
- 卡牌濒危警告提示

#### 5.2.2 NPC对话界面

**布局**:
- 左边顶部: NPC头像、名称、专长领域
- 左边中间: 对话气泡区域（一问一答形式）
- 左边底部: 用户输入框、快捷问题按钮
- 右边: 冒险者日记（笔记区）
- 右边底部: 结束学习，提交笔记

**交互**:
- 用户输入或点击快捷问题
- NPC气泡展示回答
- 支持代码块展示
- 结束学习后提交笔记自动弹出"获得卡牌"动画

#### 5.2.3 Boss战界面

**布局**:
- Boss信息区: 名称、难度星级、弱点标签
- 题目区: Markdown格式题目描述
- 卡牌选择区: 用户选择要使用的卡牌
- 战斗准备区: 代码输入框/选项列表，提交按钮

**交互**:
- 点击选择卡牌
- 输入答案/选择选项
- 提交后即时反馈结果

#### 5.2.4 卡牌工坊

**布局**:
- 左侧: 领域筛选标签
- 右侧: 卡牌网格展示
- 卡牌卡片: 名称、耐久度条、关联笔记入口

**交互**:
- 点击卡牌查看详情
- 点击"复习"进入复习流程
- 濒危卡牌高亮提醒

### 5.3 游戏难度设置

游戏难度影响以下参数：

| 难度等级 | 耐久度变化率 | Boss掉卡率 | 每日任务数量 |
| -------- | ------------ | ---------- | ------------ |
| **简单** | 降低50%      | +15%       | 3个          |
| **普通** | 标准         | 标准       | 5个          |
| **困难** | 增加50%      | +30%       | 8个          |

**说明**:
- **耐久度变化率**: 影响复习/Boss成功时耐久度提升幅度，以及未复习/失败时耐久度下降幅度
- **Boss掉卡率**: 击败Boss后掉落新卡牌的基础概率（可通过成就/连续成功等加成）
- **每日任务数量**: 每天推送的复习+Boss任务总数

## 6. 技术架构

### 6.1 技术选型

| 组件         | 技术                       | 说明                   |

## 8. 设计原则总结

1. **游戏化优先**: 每个功能都要考虑如何让用户感到"有趣"和"有成就感"
2. **不伤害用户**: 不出超出用户能力的题目，保护学习积极性
3. **遗忘曲线为王**: 复习机制是产品的核心价值，必须科学执行
4. **即时反馈**: 每次操作都要有及时的视觉/交互反馈
5. **简洁不简单**: 界面简洁，但背后的机制要有深度
